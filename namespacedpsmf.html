Detailed Description

This file defines the core for the Dynamic Publisher Subscriber Mock Framework.

The DPSMF object requires:

  • 'base_d': Base path of project to recursively search (Default: pwd)
  • 'out_d': Output path for the documentation (Default: pwd)
  • 'sim': Flag indicating the project is being built for simulation (Default false)

At this point, the program shall execute, recursively searching directories for appropriately named YAML files. The appropriate boilerplate code shall generate with the specified language at the location that the file was located.

The allowed languages are: Python, Rust, C/C++.

After all of the files have been generated, a documentation file of the API generated will be created and placed in the out_d directory.
